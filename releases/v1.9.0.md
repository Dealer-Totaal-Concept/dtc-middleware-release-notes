# DTC Middleware v1.9.0

**Release Date:** 2026-06-03T20:08:50Z

## Release Notes

## Release scope

This draft release covers the production candidate from  onward. The  release was a test release and is intentionally skipped as the comparison baseline.

## Highlights

- Added the Consumer Credit (CK) funnel foundation, financier provider integration, frontend-facing endpoints, and operational routes for deal updates and cancellations.
- Added VKM calculation support, including the frontend endpoint contract, HubDB-backed variables, Excel v6 alignment, and improved amount parsing for form submissions.
- Expanded BKR and dealer portal automation, including BKR object handling, lead creation, salary-slip upload support, mileage handling, and safer trade-in registration.
- Improved HubSpot and provider reliability with client-profile handling, owner lookup fallback, portal-user availability caching, BNP polling fixes, Defam polling backoff, and cleaner operational controls.
- Hardened runtime and release operations with dotenvx-based Heroku configuration, Node 24 CI updates, safer logging, release-candidate health checks, and the GitHub-to-Heroku release flow.

## CK funnel and VKM

- Introduced shared CK DTOs, infrastructure, provider integration, and application-routing support. (#306, #329, #438)
- Added CK frontend funnel endpoints for deal creation, updates, cancellation, active-deal checks, and request-contract alignment. (#279, #387, #442, #449, #450)
- Added the VKM calculator endpoint and aligned calculations with the Excel v6 reference model, including HubDB variable-table configuration. (#381, #412, #413, #433)
- Improved form compatibility and validation by parsing submitted euro amount strings and tightening request payload behavior. (#417)

## BKR, dealer portal, and lead flow

- Added BKR object validation, creation, deletion, processor integration, and proposal mapping coverage. (#282, #298, #311, #382, #416)
- Added fallbacks for BKR  and  responses so downstream behavior is more predictable. (#446)
- Added dealer portal lead creation and calculator-entry updates, including mileage handling and typo fixes. (#418, #439, #445, #448)
- Added salary-slip upload support, connected income and expense data to contacts, and only registered trade-in objects when the balance is known. (#428, #432, #440)

## Providers, HubSpot, and operational reliability

- Improved BNP integration behavior across release handling, stipulation upload stability, polling cache handling, supplier fallback data, and sensitive log output. (#264, #265, #269, #281, #305, #375)
- Added debt-consolidation and redemption support, contract-id support in financier requests, and quote-result DTO improvements. (#267, #268)
- Reduced noisy or risky provider logging, including raw Axios error output and token exposure risks. (#299, #301)
- Added Defam polling backoff and better HubSpot owner/profile behavior for portal-user routing and operational controls. (#384, #388, #393, #395, #396, #403, #419, #420)

## Runtime, release, and CI/CD

- Moved Heroku runtime configuration to dotenvx and added encrypted production configuration support. (#378, #421, #422, #429, #441)
- Updated CI to Node 24 and refreshed GitHub Actions behavior. (#430)
- Added the GitHub release workflow, Heroku release-candidate app validation, automatic production promotion after passing health checks, and clearer action naming. (#447)

## Security, stability, and tests

- Improved Redis resilience, transient-error handling, timeout behavior, and log severity around operational failures. (#286, #291, #303, #304, #307, #308, #309, #312)
- Addressed the Axios TLS listener leak by updating Axios. (#313)
- Expanded focused test coverage across CK, BKR, provider, release, and documentation-sensitive behavior. (#257, #266, #273, #294, #314, #377, #416, #437)
